# 🚀 AccountIA Sprint 2 - Guía de Instalación para Windows\n\n## ⚡ Inicio Rápido para Windows\n\n### 1. **Setup Automático**\n```powershell\n# Ejecutar setup completo con PowerShell\nnpm run setup:sprint2:windows\n\n# O si prefieres verificar primero\nnpm run verify:sprint2:windows\n```\n\n### 2. **Instalación Manual (si el setup automático falla)**\n\n#### Paso 1: Instalar Dependencias\n```powershell\n# Dependencias root\nnpm install\n\n# Dependencias frontend\ncd frontend\nnpm install\ncd ..\n\n# Dependencias backend\ncd backend\npip install -r requirements.txt\ncd ..\n```\n\n#### Paso 2: Configurar Archivos de Entorno\n```powershell\n# Copiar archivos de ejemplo\ncopy .env.example .env\ncopy backend\\.env.example backend\\.env\n```\n\n#### Paso 3: Crear Directorios\n```powershell\n# Crear directorios necesarios\nmkdir backend\\media\nmkdir backend\\staticfiles\nmkdir backend\\mock_storage\nmkdir backend\\logs\nmkdir logs\nmkdir backups\n```\n\n### 3. **Iniciar Servicios**\n\n#### Opción A: Docker (Recomendado)\n```powershell\n# Iniciar servicios con Docker\nnpm run docker:up\n\n# En terminales separadas:\nnpm run backend:dev\nnpm run frontend:dev\n```\n\n#### Opción B: Sin Docker (Solo para desarrollo frontend)\n```powershell\n# Terminal 1: Solo frontend\nnpm run frontend:dev\n\n# Terminal 2: Backend (requiere PostgreSQL y Redis locales)\nnpm run backend:dev\n```\n\n### 4. **Verificar Instalación**\n```powershell\n# Verificar que todo esté funcionando\nnpm run verify:sprint2:windows\n\n# Health check de servicios\nnpm run health\n\n# Ver versión\nnpm run version\n```\n\n## 🔧 Resolución de Problemas en Windows\n\n### Error: \"execution policy\"\n```powershell\n# Cambiar política de ejecución (como administrador)\nSet-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser\n\n# O ejecutar directamente\npowershell -ExecutionPolicy Bypass -File scripts/setup_sprint2.ps1\n```\n\n### Error: \"python no encontrado\"\n```powershell\n# Instalar Python desde Microsoft Store o python.org\nwinget install Python.Python.3.11\n\n# O usar py launcher\npy --version\npy -m pip install -r backend/requirements.txt\n```\n\n### Error: \"Node.js no encontrado\"\n```powershell\n# Instalar Node.js\nwinget install OpenJS.NodeJS\n\n# O descargar desde nodejs.org\n```\n\n### Error: \"Docker no encontrado\"\n```powershell\n# Instalar Docker Desktop\nwinget install Docker.DockerDesktop\n\n# O descargar desde docker.com\n```\n\n### Error: \"npm install falla\"\n```powershell\n# Limpiar cache y reinstalar\nnpm cache clean --force\nrmdir /s node_modules\nnpm install\n```\n\n### Error: \"Puerto ya en uso\"\n```powershell\n# Encontrar qué está usando el puerto\nnetstat -ano | findstr :3000\nnetstat -ano | findstr :8000\n\n# Terminar proceso si es necesario\ntaskkill /PID <PID> /F\n```\n\n## 📋 Comandos Útiles para Windows\n\n### Desarrollo Diario\n```powershell\n# Iniciar desarrollo\nnpm run docker:up\nnpm run backend:dev      # En nueva terminal\nnpm run frontend:dev     # En nueva terminal\n\n# Verificar estado\nnpm run health\nnpm run logs\n\n# Al finalizar\nnpm run docker:stop\n```\n\n### Debugging\n```powershell\n# Ver logs específicos\nnpm run logs:backend\nnpm run logs:frontend\nnpm run logs:postgres\nnpm run logs:redis\n\n# Limpiar y reiniciar\nnpm run docker:clean\nnpm run setup:sprint2:windows\n```\n\n### Testing\n```powershell\n# Ejecutar tests\nnpm run test\nnpm run test:backend\nnpm run documents:test\n\n# Linting y formato\nnpm run lint\nnpm run format\n```\n\n### Base de Datos\n```powershell\n# Migraciones\nnpm run db:migrate\nnpm run db:makemigrations\n\n# Shell de base de datos\nnpm run db:shell\n\n# Crear superusuario\nnpm run create:superuser\n```\n\n## 🌐 Acceso a la Aplicación\n\nUna vez que los servicios estén corriendo:\n\n- **Frontend**: http://localhost:3000\n- **Backend API**: http://localhost:8000\n- **Admin Django**: http://localhost:8000/admin\n- **Health Check**: http://localhost:8000/health/\n- **API Docs**: http://localhost:8000/api/docs/\n\n## 📊 Verificación de Servicios\n\n### PowerShell Health Check\n```powershell\n# Verificar servicios manualmente\n# Backend\nInvoke-WebRequest -Uri \"http://localhost:8000/health/\" -Method Get\n\n# Frontend\nInvoke-WebRequest -Uri \"http://localhost:3000\" -Method Get\n\n# Verificar puertos en uso\nnetstat -an | Select-String \":3000|:8000|:5432|:6379\"\n```\n\n### Docker Status\n```powershell\n# Ver contenedores corriendo\ndocker ps\n\n# Ver logs de
